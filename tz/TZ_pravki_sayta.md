## Задачи

### П1 — Слайдер ближайших игр: некорректная отрисовка плашки с датами
**Тип:** баг  
**Файлы:** `main/core/theme/default/views/home/home.html` (строки 80–237), связанный JS/CSS Owl Carousel

**Описание:**  
При прокрутке карусели вправо плашка с датами (`schedule__date`) рендерится частично — "половина как будто прогружается".

**Вероятные причины:**
- CSS `overflow: hidden` или `clip` на родителе карусели обрезает абсолютно позиционированную плашку.
- Owl Carousel не пересчитывает высоту/положение элементов при смене слайда (событие `translated` не обновляет стили).

**Критерий готовности:** плашка с датами полностью видна и корректно отрисована при прокрутке в любую сторону.

### П3 — Регистрация: некоторые команды попадают на пустую страницу (ПРИОРИТЕТ)
**Тип:** критический баг  
**Файлы:** `main/home/views.py` → `game_callback()` (строки 81–199), `main/home/forms.py` → `GameOrderForm` (строки 189–294)

**Описание:**  
При отправке формы регистрации команда попадает на пустую страницу вместо редиректа на `/?reserve=true/false`.

**Вероятные причины:**
1. Исключение в `game_callback()` до блока `redirect` — Django возвращает 500 без шаблона ошибки (пустая страница).
2. Игра (`Games`) с переданным `game_id` не найдена → `get()` бросает `DoesNotExist`, исключение не поймано.
3. Ошибка при отправке сообщения в Telegram (если Telegram-бот недоступен) — исключение прерывает выполнение до редиректа.
4. CSRF-ошибка при некоторых способах submit (редко).

**Что сделать:**
1. Добавить `try/except` вокруг `Game.objects.get(id=game_id)` с редиректом на `/?error=true`.
2. Обернуть отправку в Telegram в `try/except` — ошибка Telegram не должна блокировать сохранение заявки.
3. Убедиться, что ошибка формы тоже редиректит (не возвращает пустой Response).
4. Проверить логи сервера, чтобы подтвердить реальную причину.

**Критерий готовности:** при любом сценарии (невалидные данные, Telegram недоступен, игра не найдена) пользователь редиректируется, а не видит пустую страницу.
