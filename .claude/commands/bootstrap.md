# Bootstrap проекта

Запусти **один раз** на проекте. Пропускай шаги, где файл-цель уже существует и заполнен (без `<!-- TODO -->` плейсхолдеров).

## Фаза 0 — определить тип проекта

Проверь корень рабочей директории: исходники, конфиги пакетов, существующие директории кода.

- **Greenfield**: директория пустая либо содержит только `.claude/`, `.cursor/`, `docs/`, `.git/`, `README.md` — без исходников и dependency-файлов.
- **Legacy**: есть исходники, `package.json` / `requirements.txt` / `go.mod` / `Cargo.toml` / `composer.json`, или непустая структура директорий.

Сообщи пользователю детектированный тип до продолжения.

## Фаза 1 — контекст проекта (параллельные `Explore`-субагенты)

### Для обоих типов

- **`.cursorignore` / `.gitignore`**: если отсутствуют — сгенерируй из `.gitignore` (если есть) плюс стандартные AI-исключения: lockfiles, билд-артефакты, скомпилированные файлы, медиа, секреты.
- **`docs/INDEX.md`**: если отсутствует — просканируй `docs/` на `.md` и корневой `README.md`. Создай оглавление со ссылками. Отсутствующие цели — помечай `<!-- TODO: создать -->`. Если есть `docs/skills-index.md` — включи ссылку.

### Только Greenfield

- **`docs/PROJECT.md`**: создай шаблон с секциями (Стек, Структура, Точки входа, Соглашения, Ограничения) и `<!-- TODO -->` плейсхолдерами — пользователь заполнит по мере развития.

### Только Legacy

- **`docs/PROJECT.md`**: просканируй dependency-файлы, дерево директорий, точки входа, паттерны кода. Заполни шаблон детектированными данными. Отчитайся о находках.

## Фаза 2 — модульные CLAUDE.md (только legacy)

**Полностью пропускай для greenfield.**

**Триггер**: в `main/<app>/` (или эквивалентных модулях) нет `CLAUDE.md`. Если модульные `CLAUDE.md` уже существуют — пропусти.

Используй субагента `Explore`:

1. **Детект модулей** — анализ верхнего уровня и `src/` / `main/`, границы языков/фреймворков, директории с ≥ 5 связанных исходников.
2. **Глубокое чтение каждого модуля** — 3–5 репрезентативных файлов, импорты, нейминг, обработка ошибок, тесты.
3. **Сгенерируй `<module>/CLAUDE.md`** на модуль:
   - Ключевые пути (со ссылками markdown на файлы внутри модуля)
   - Ответственность и границы
   - Паттерны и кросс-модульные контракты
   - Не длиннее **40 строк**
4. **Отчёт** пользователю: список созданных `CLAUDE.md` с одной строкой описания.

## Фаза 3 — самоочистка

После завершения всех фаз:

1. Удали этот файл `.claude/commands/bootstrap.md` — он свою роль выполнил.
2. Сообщи пользователю, что очистка сделана.

## Экономия токенов

- Тяжёлое сканирование — в `Explore`-субагентах. Основной контекст остаётся чистым.
- Модульные `CLAUDE.md` подхватываются Claude Code автоматически при работе с файлами внутри директории.
- Правила ссылаются на пути и паттерны, не дублируют содержимое `docs/PROJECT.md`.
- Тривиальные модули (≤ 3 файла, простой CRUD) — пропускай создание `CLAUDE.md`.
